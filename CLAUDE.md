## Project Overview

**Comunali** — Real-time Italian municipal election vote counting and visualization app. Full-stack monorepo with a Quarkus (Java 17) backend and React (Vite) frontend, backed by PostgreSQL.

## Development Commands

### Backend (from `backend/`)
```bash
mvn quarkus:dev                    # Dev server with hot reload (port 8080)
mvn clean package                  # Build uber-JAR
mvn test                           # Run tests
mvn -Pnative package               # GraalVM native build
```

### Frontend (from `frontend/`)
```bash
npm install                        # Install dependencies (no lockfile)
npm run dev                        # Vite dev server (port 5173, proxies /api → :8080)
npm run build                      # Production build → dist/
npm run lint                       # ESLint
```

### Full stack (from root)
```bash
docker compose up                  # Starts postgres + backend (dev)
docker compose --profile prod up   # Starts postgres + backend + nginx frontend
```

## Architecture

### Monorepo Layout
- `backend/` — Quarkus 3.8.2 REST API (Java 17, Maven)
- `frontend/` — React 18 SPA (Vite 5, Tailwind CSS 3, Recharts)
- `docker-compose.yml` — Local dev (PostgreSQL 15 + backend + optional nginx)
- `.github/workflows/deploy.yml` — CI/CD (build + deploy to Render/Vercel)

### Backend Patterns
- **Panache Active Record** — Entities extend `PanacheEntity`, no DAO/repository layer
- **JAX-RS Resources** — REST endpoints in `resource/` package, one per entity
- **JWT Auth** — SmallRye JWT with RSA 2048-bit keys, 8h expiry. Keys in resources/ (dev only)
- **DTOs** — Request/response objects in `dto/` package
- **Role-based access** — `@RolesAllowed` annotations. Roles: `ADMIN`, `GESTORE_CANDIDATI`, `GESTORE_LISTE`, `GESTORE_VOTI`
- **Profiles** — `dev` (default, SQL logging), `prod` (env-var JWT keys, schema drop-and-create on first run)

### Frontend Patterns
- **Pure JavaScript** (JSX, no TypeScript)
- **React Context** for auth state only (`AuthContext.jsx`), component-level `useState` elsewhere
- **Axios client** (`api/client.js`) with JWT interceptor and 401 auto-logout
- **One API module per resource** in `src/api/`
- **PrivateRoute** component with role checking
- **Dashboard** auto-refreshes every 30 seconds

### Database
- PostgreSQL 15, schema auto-generated by Hibernate
- Initial data loaded from `backend/src/main/resources/import.sql`
- Default credentials: `admin` / `admin123`

## Domain Model

Italian municipal election entities:
- **Sezione** — Polling station (20 sections, each with voter eligibility count)
- **ListaElettorale** — Electoral list/party (with color for UI)
- **CandidatoSindaco** — Mayor candidate (M2M with lists for coalitions)
- **CandidatoConsigliere** — Council candidate (belongs to one list)
- **VotoSezione** — Vote counts per section per list (supports *voto disgiunto*)
- **PreferenzaConsigliere** — Preference votes for individual councillors

## Deployment

- **Database**: Neon.tech (serverless PostgreSQL)
- **Backend**: Render.com (free tier, cold starts after 15min inactivity)
- **Frontend**: Vercel (root directory: `frontend`)
- GitHub Actions deploys on push to `main` (conditional on secrets being configured)

### Required GitHub Secrets
`RENDER_DEPLOY_HOOK_URL`, `VERCEL_TOKEN`, `VERCEL_ORG_ID`, `VERCEL_PROJECT_ID`, `VITE_API_BASE_URL`

### Backend Env Vars (prod)
`DB_URL`, `DB_USER`, `DB_PASSWORD`, `JWT_PUBLIC_KEY`, `JWT_PRIVATE_KEY`, `CORS_ORIGINS`
