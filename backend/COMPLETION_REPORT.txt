    -d '{"username":"admin","password":"admin123"}'

BUILD PRODUCTION
  $ mvn clean package -Dquarkus.package.type=uber-jar

BUILD NATIVE (Richiede GraalVM)
  $ mvn clean package -Pnative
  $ ./target/comunali-backend-1.0.0-runner

DEPLOY DOCKER

================================================================================
                    MIGRATION FROM DEVELOPMENT
================================================================================

TO DEPLOY TO PRODUCTION:

1. ENVIRONMENT SETUP
   - Configure PostgreSQL database (or use managed service)
   - Generate new JWT keys (optional, current ones in repo)
   - Set environment variables: DB_USER, DB_PASSWORD, DB_URL

2. BUILD
   - Run: mvn clean package -Dquarkus.package.type=uber-jar
   - Output: target/comunali-backend-1.0.0-runner.jar

3. DEPLOY
   Option A: Docker
     - docker build -t comunali-backend:1.0.0 .
     - docker push to registry
     - Deploy with Docker Compose or Kubernetes

   Option B: Render.com
     - Connect GitHub repository
     - Configure environment variables
     - Deploy (auto build + run)

   Option C: VPS
     - Copy JAR to server
     - Run: java -jar comunali-backend-1.0.0-runner.jar
     - Reverse proxy with Nginx/Apache

4. POST-DEPLOYMENT CHECKS
   - Health check: curl http://localhost:8080/q/health
   - Login test: curl -X POST http://localhost:8080/api/auth/login ...
   - Dashboard: curl http://localhost:8080/api/dashboard/riepilogo ...
